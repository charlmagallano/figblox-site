Prefixes

Recipes for real Figma layouts—then the full prefix list when you need it.

Prefixes tell Figblox what to create. Put them at the start of the layer name.

Use this format:
.prefix Layer Name

The text after the last prefix becomes the layer name in Roblox.
Example: .textbutton Shop creates a TextButton named Shop.

Most layers do not need a prefix. If your frame, text, or image already imports as the right Roblox type, leave the name clean—prefixes are for steering behavior when the default is not enough.

When should I use prefixes?

  • If the layer already imports correctly, use no prefix.
  • If you need a specific Roblox class, use a role prefix such as .textbutton, .imagebutton, .scrollv, .scrollh, or .canvas.
  • If the layer should become a baked PNG, use .raster.
  • If a child should become the actual Roblox parent for its siblings, use .parent on that direct child.
  • If the layer is a state variant for import or merge workflows, use .hover, .pressed, .toggled, or .disabled.
  • If the layer is only a Figma helper (guides, notes, scratch geometry), use .ignore.

Prefix recipes

These patterns mirror how designers structure buttons, lists, and panels in Figma—prefixes line up with those jobs in Roblox.

Button with image background and text

Recommended structure:

.imagebutton Close
  X
  .parent .raster Background

The wrapper sets the final object to an ImageButton. .parent makes Background the real Roblox parent/host. .raster bakes the visual background into one PNG. X stays as a child inside the button.

Image button backgrounds. Pair .parent with .raster on the background layer when the artwork should bake to an image and that layer should host the rest of the hierarchy.

Avoid:

.imagebutton Close
  .parent X
  .parent Background

Only one direct child in a group should use .parent. More than one competing parent tag is ambiguous.

Simple text button

.textbutton Play

Use this when the button can be native Roblox text without a custom image background.

Inventory / slot scroll list

.scrollv SlotScroll
  Slots
    Slot 1
    Slot 2

Use .scrollv for vertical scrolling and .scrollh for horizontal scrolling. For both axes on the same frame, add both .scrollh and .scrollv—order does not matter—or set Figma overflow to Both and let Figblox detect it (see Auto Layout). Auto layout inside the scroll content can become list or grid behavior.

Grid / wrapped slots

.scrollv InventoryScroll
  Slots
    Slot 1
    Slot 2
    Slot 3

Use Figma auto layout with horizontal flow and wrap for grids. Figblox imports that layout into Roblox UI layout objects.

Complex art or effects

.raster Card Art

Use .raster when the layer has gradients, masks, blur, overlapping shapes, or effects that should stay visually faithful as a single PNG.

Parent override for cards or buttons

.frame Card
  Title
  .parent Background

.parent on a direct child makes that child become the Roblox parent; siblings are placed inside it. Use this when the visual background should be the real container.

One parent tag per group. Only one direct child under a frame or group should use .parent.

State variants

.imagebutton Buy
.pressed Buy
.hover Buy
.disabled Buy

These layers are separate frames or components in Figma that share the same Roblox-facing name. State prefixes mark which variant is hover, pressed, disabled, and so on for import and matching workflows.

Ignore helper layers

.ignore Guides

Use .ignore for guides, measurement lines, notes, or anything that should not export to Studio.

No prefix needed

Some Figma layers already export to the right Roblox type on their own.

  • Text exports as a TextLabel
  • Frames, groups, components, instances, and plain rectangles export as a Frame
  • Vectors, shapes, and image-filled frames or rectangles export as an ImageLabel
  • Anything else exports as a Frame by default

Corners, strokes, gradients, auto layout, padding, and similar Figma properties can also create Roblox UI objects automatically, like UICorner, UIStroke, UIGradient, UIListLayout, UIPadding, and UIAspectRatioConstraint.

Prefix Tags panel

The Figblox Figma plugin includes a Prefix Tags panel for editing layer names without typing prefixes by hand. Select a layer in the canvas, then use the panel to build the prefix stack and the Roblox-facing name.

  • Layer shows the combined prefix string and the layer title (for example .raster Fill).
  • Role picks object-type prefixes such as Frame, Text Label, Scroll V, Scroll H, Scroll XY, Canvas, Viewport, and related options.
  • States adds interaction prefixes such as Hover, Pressed, Toggled, and Disabled.
  • Tags adds utility prefixes such as Gray, Parent, Lock, and Ignore.

Raster in Tags toggles the .raster prefix: on adds the raster tag, off removes it. See Raster for when rasterizing makes sense and how matching behaves.

Scroll XY in Role targets two-axis scrolling—the same situation covered by putting both .scrollh and .scrollv on one layer (see Combining prefixes).

Prefer Prefix Tags in Figma when you can—it reduces typos and keeps prefix stacks consistent compared to editing raw layer names.

Combining prefixes

  • Prefixes can be stacked on one layer.
  • Order usually does not matter—Figblox applies the same tags regardless of sequence.
  • Use only one role-style prefix per layer (for example do not combine .textlabel with .imagebutton).
  • Use only one state prefix per layer (.hover and .pressed on the same layer conflict).
  • Only one direct child in a group should use .parent.
  • .ignore wins: the layer is skipped, so other tags on that layer do not affect export.
  • Do not combine .lock with .raster—they pull export behavior in different directions.
  • For scrolling frames, .scrollh sets horizontal scrolling and .scrollv sets vertical scrolling. When both appear on the same layer, you get two-axis scrolling; order does not matter.

Good stacks:

.parent .raster Background
.raster .parent Background
.imagebutton .raster Icon Button
.gray .raster Disabled Icon

Avoid:

.textlabel .imagebutton Button    ← two roles; pick one
.hover .pressed Button              ← two states; split into separate variant layers
.lock .raster Art                  ← conflicting export behavior

Full prefix reference

Object prefixes

.frameForce a Frame when you want the role explicit (many plain frames already become Frames without it).
.textlabelStatic text blocks and labels.
.textbuttonClickable buttons using native Roblox text styling.
.textboxText entry fields.
.imagelabelNon-interactive images and icons.
.imagebuttonUse for clickable image-based buttons.
.imageframeSame end result as .imagelabel—an ImageLabel in Roblox.
.scrollvVertical scrolling regions (ScrollingFrame).
.scrollhHorizontal scrolling regions (ScrollingFrame).
.canvas or .canvasgroupCanvasGroup for layered effects and group transparency.

Other prefixes

.gray or .grayscaleExport image or raster output as grayscale.
.hoverMarks this layer as the Hover variant for state matching.
.clicked or .pressedMarks this layer as the Pressed variant.
.toggledMarks this layer as the Toggled variant.
.disabledMarks this layer as the Disabled variant.
.rasterBake this layer or subtree into a PNG.
.parentMake this direct child become the actual Roblox parent for its siblings.
.ignoreSkip this layer during export.
.lockApplies lock behavior; combine with other prefixes except .raster.

Examples

.textlabel Health Bar
.imagebutton Play Button
.scrollv Leaderboard
.scrollh .scrollv Map Panel
.gray .imagelabel Title
.pressed .imagelabel OK
.parent .raster Background
.gray .scrollh .raster Card
