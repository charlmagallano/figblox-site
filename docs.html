Getting Started

Overview

Figblox is a two-plugin workflow: export structured UI data from Figma, then import it into Roblox Studio with merge-aware updates. You keep iterating in Figma while preserving the Studio-side wiring and scripts you added after import.

Install Both Plugins

Install the Figblox exporter from the Figma Community and the Figblox importer from the Roblox Creator Store. You need both — Figma produces the JSON bundle and assets; Studio consumes them inside your place.

Tip: Pin both plugins for faster iteration during UI polish passes.
  • Figma Community (placeholder)
  • Roblox Creator Store (placeholder)

Basic Flow

  1. Design UI in Figma using frames, components, and auto layout.
  2. Run Export in the Figblox Figma plugin to generate JSON and image assets.
  3. Open the Figblox Studio plugin, paste JSON (and attach the ZIP if prompted).
  4. Review the preview diff, then Confirm to merge changes into your ScreenGui.

Prefixes

Object Prefixes

Object prefixes appear at the start of a layer name and tell Figblox which Roblox instance type to create. If you omit a prefix, Figblox defaults to a Frame.

.frameFrame
.textlabelTextLabel
.textbuttonTextButton
.textboxTextBox
.imagelabelImageLabel
.imagebuttonImageButton
.scrollingframe / .scrollvVertical ScrollingFrame
.scrollingframex / .scrollhHorizontal ScrollingFrame
.viewportframeViewportFrame
.canvas / .canvasgroupCanvasGroup

Modifier Prefixes

Modifiers stack with object prefixes and change export behavior — rasterization, skipping, lock hints, or interaction state markers.

.rasterRenders the layer as a PNG
.gray / .grayscaleExports as grayscale PNG
.ignoreSkips layer during export
.lockApplies lock behavior
.parentMakes this child the Roblox parent of its group
.hover / .clicked / .pressed / .toggled / .disabled State markers

Combining Prefixes

Combine modifiers left-to-right with the object prefix at the start of the name. Order can affect how Figblox interprets raster and scroll hints — keep the object type prefix first, then modifiers.

.textlabel Title
.gray .raster .imagelabel Badge
.scrollh .raster Banner
Note: If Figblox warns about an ambiguous combination, simplify the layer name or split content into child layers with explicit prefixes.

Parent Override

The .parent modifier marks a child layer as the Roblox parent for its sibling subtree during import. Use it when you need a specific instance under a group to become the container for generated children.

Auto Layout

Scroll Containers

In Figma, set overflow on frames to vertical or horizontal where appropriate. Figblox maps eligible frames to ScrollingFrame with direction and clipping aligned to your auto layout choices.

Auto Layout → Roblox

Auto layout generally maps to UIListLayout, UIGridLayout, and UIPadding depending on structure. Spacing, padding, and alignment are carried into layout objects so your hierarchy stays responsive after To Scale runs on import.

Images

How Export Works

Raster layers and configured image fills export as PNGs packaged with your JSON. Figblox deduplicates identical bitmaps and records stable names so Studio can link assets consistently across re-imports.

Importing Images

When you confirm an import, Figblox matches exported filenames to Roblox assets where possible. If a decal or image object exists in your place and names align, the importer links without manual assignment.

Important: For automatic linking, keep asset names predictable and consistent between exports.

Re-importing Images

Updated PNGs from Figma replace prior associations only where the merge detects a change. Non-image Studio tweaks on instances are preserved according to the smart merge rules.

Raster layers

Make Raster

Tag layers with .raster to force bitmap export. Use this for effects Roblox text and vector shapes cannot easily replicate from pure GUI instances.

Clear Raster

Remove the .raster modifier to return a layer to normal vector export behavior on the next run. Re-export to refresh the bundle.

When to Use Raster

Prefer native instances when you need live text or scalable UI. Rasterize when fidelity matters more than editability — soft shadows, complex blends, or tightly designed badges.

Import Process

Preview

Each import starts with a diff-style preview so you can see what would change before it touches your hierarchy.

[+] New layer [~] Changed [=] Already in sync

New layers appear as additions. Changed layers highlight differences from Studio. In-sync layers require no update for that import.

Confirm

Confirm applies the merge, runs linking for images, and executes To Scale, Lock Text, and Lock Stroke as part of the same pass — you do not need to trigger those tools manually for a standard import.

Re-importing

Re-importing merges intelligently: updates from Figma apply where Figblox detects changes, while unrelated Studio edits remain on instances that are still tied to their originating layers.

Remove Deleted Layers

By default, layers removed in Figma are not deleted in Studio. Enable “Remove deleted layers on import” when you want deletions in Figma to prune matching instances on confirm.

Warning: Pruning can remove instances you still need if names changed or mappings drifted — preview carefully.

Tools

To Scale

Converts pixel measurements to scale-friendly values so UI responds predictably across viewport sizes. Confirm runs this during import.

Lock Text

Preserves text sizing behavior when parent containers scale — helpful for keeping labels crisp and readable.

Lock Stroke

Keeps UIStroke thickness stable instead of stretching with parent scale during responsive adjustments.

Link Images

Attempts to bind exported PNG references to existing Roblox assets by name and path hints, reducing manual decal wiring.

Reset Tools

Reset returns tool toggles and transient importer settings to defaults for the current session — useful when experimenting with partial imports.

Inspector

The inspector surfaces per-instance mapping metadata and warnings from the last preview or import, including unresolved images or ambiguous prefixes.

Components

Make Button

Converts a Frame or ImageLabel into an interactive button with appropriate input hooks while preserving children where possible.

Make TextBox

Turns a TextLabel into an editable TextBox configured for player input.

Make ScrollView

Wraps an existing Frame with a ScrollingFrame and moves content into a scrollable subtree.

Make Toggle

Builds a toggle control from a root Frame with On/Off child frames that the plugin wires into a single boolean control.

Workflow

Full Workflow Guide

  1. Structure Figma layers with prefixes early — rename passes are cheaper before export.
  2. Export JSON + assets whenever the design meaningfully changes.
  3. Import into a staging ScreenGui first if you are unsure about merge scope.
  4. Script against stable instance names; avoid one-off renames in Studio that fight Figma.
  5. Re-export and merge often — smaller deltas are easier to read in preview.

History

Figblox tracks recent imports in the Studio plugin so you can compare bundle timestamps and quickly roll back to a prior JSON paste if needed. This history is local to your machine and place session — it is not uploaded to Figblox servers.
